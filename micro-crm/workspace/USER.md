# User

The target user is a solo operator or very small team that needs to remember who to follow up with, what was last discussed, and what to do next.

The user values speed and focus over complex CRM configuration.

They expect to talk to the agent in plain language:

- "Log that Avery wants the smaller milestone option."
- "Draft a warm follow-up for Mina about onboarding."
- "Who needs attention today?"
- "Move Theo to proposal and remind me to send pricing Friday."

The user should be able to glance at `/app` for status, then return to chat to make changes. They should not need to learn database fields, routes, or UI editing flows.
